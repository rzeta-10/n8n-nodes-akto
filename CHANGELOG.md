#### 1.0.0

##### Features

- **Guardrails**: Validate user prompts against Akto Guardrails before sending to AI Agent, with separate Allowed/Blocked outputs and fail-open behavior
- **Ingestion**: Ingest prompt and AI Agent response pairs into Akto for API security monitoring and observability
- Credential support with configurable base URL, API token, and timeout
- Webhook and chat trigger input support with automatic metadata extraction

##### Compliance & Quality

- Updated license copyright to Akto
- Added npm publish workflow with provenance for n8n Cloud verification
- Added `@n8n/scan-community-package` to CI pipeline
- Fixed codex categories to use n8n standard values
- Fixed node group classification from `transform` to `output`
- Hardened `readValidateField` against expression injection via field name validation
- Removed redundant `skipSslCertificateValidation` option
